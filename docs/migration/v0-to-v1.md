# Migration Guide: v0.x → v1.0

Forge v1.0 is a breaking release. It removes 6 commands, renames 7 commands, removes the `mode` config field, and deletes fast-mode dead code. This guide covers everything you need to update.

---

## Minimum upgrade path

| Your current version | Upgrade path |
|---|---|
| v0.40.0 – v0.51.x | Upgrade directly to v1.0.0 |
| v0.12.x – v0.39.x | Update to v0.51.x first, then upgrade to v1.0.0 |
| Below v0.12.x | Not supported. Contact support. |

To upgrade:

```bash
/plugin install forge@skillforge   # or forge@forge for canary
/forge:update                      # applies migration entries to your project
```

## Command renames (7)

All 7 renamed commands still work — old names return a redirect error with the new name.

| Old command | New command | Notes |
|---|---|---|
| `/forge:sprint-intake` | `/forge:new-sprint` | Same behavior |
| `/forge:sprint-plan` | `/forge:plan-sprint` | Same behavior |
| `/forge:retrospective` | `/forge:retro` | Same behavior |
| `/forge:regenerate` | `/forge:rebuild` | Same behavior; adds `--enrich` flag |
| `/forge:store-query` | `/forge:search` | Same behavior |
| `/forge:store-repair` | `/forge:repair` | Same behavior |
| `/forge:quiz-agent` | `/forge:check-agent` | Same behavior |

**Project-scoped commands** (generated by init) are also renamed:
- `/project:sprint-intake` → `/project:new-sprint`
- `/project:sprint-plan` → `/project:plan-sprint`
- `/project:retrospective` → `/project:retro`

Run `/forge:rebuild commands` after upgrade to regenerate command wrappers with the new names.

## Command removals (6)

Removed commands return a clear error message with the replacement. They are no longer registered in the command list.

| Removed command | Replacement | How to migrate |
|---|---|---|
| `/forge:update-tools` | `/forge:update` | Schema refresh is now automatic on plugin update |
| `/forge:materialize` | *(removed)* | Fast-mode stubs no longer exist. See [Mode field removal](#mode-field-removal) |
| `/forge:enhance` | `/forge:rebuild --enrich` | Run `rebuild --enrich` for KB enrichment and drift detection |
| `/forge:calibrate` | `/forge:health --fix` | Run `health --fix` for drift detection and patch application |
| `/forge:collate` | *(internal only)* | Not user-facing. The collation process runs automatically. |
| `/forge:migrate` | `/forge:init --migrate` | Run `init --migrate` to migrate legacy stores |

## Mode field removal

The `mode` field (`fast` / `full`) has been removed from `.forge/config.json`. Fast-mode stubs no longer exist.

**What this means:**
- `/forge:init` no longer accepts `--fast` or `--full` flags
- `/forge:config mode full` is no longer a valid command
- `/forge:materialize` has been removed (there is nothing left to materialize)
- All Forge instances are now fully generated at init time

**If you had a fast-mode project:**
- Your existing stub files continue to work if they were already materialized
- Un-materialized stubs should be materialized manually before upgrading: run `/forge:materialize` on your v0.x install before upgrading, then run `/forge:update` on v1.0
- After upgrade, running `/forge:rebuild` will regenerate any missing artifacts

**Migration entry:** `forge:update` handles this automatically for projects on v0.40.0+.

The following were removed as part of the fast-mode cleanup:

- `forge/tools/ensure-ready.cjs` — deleted
- `forge/tools/__tests__/ensure-ready.test.cjs` — deleted
- `forge/init/generation/lazy-materialize.md` — deleted
- `FAST_STUB_SENTINEL` constant — removed from all command files
- Fast-mode stub detection logic — removed from `commands/rebuild.md`

These only affect Forge plugin developers. User-facing behavior is unchanged.

## New features in v1.0

### `/forge:health --fix`
Health now runs drift detection and patch application when `--fix` is passed. Previously this required `/forge:calibrate`. Every diagnostic finding maps to exactly one recommended command.

```bash
/forge:health --fix   # detect drift, propose and apply patches with confirmation
```

### `/forge:status` (new command)
Sprint and task status is now a plugin-level command, available in both the plugin and forge-cli.

```bash
/forge:status         # shows active sprint, task status counts, recent activity
```

### Pipeline step guards
Pipeline commands (`plan`, `implement`, `review-plan`, `review-code`, `validate`, `approve`, `commit`) now check task state before executing. Out-of-order execution shows a clear error with guidance.

```
× Task PROJ-S01-T03 is in state 'planned' — /forge:review-plan must complete first.
  To run the full pipeline: /forge:run-task PROJ-S01-T03
```

Use `--force` to bypass the guard.

### Revision loop visibility
Review phase output now shows `(iteration N of M)`. On the final iteration without approval, options are shown: force-approve, increase limit, restart from review.

### Post-init welcome message
`/forge:init` now prints a concise welcome block at the end of init with Tier 1 commands and next steps.

### Tomoshibi context-aware responses
`/forge:ask "what now?"` returns context-aware next steps based on project state:
- No init → suggest `/forge:init`
- Post-init → suggest `/forge:new-sprint`
- Mid-sprint → show `/forge:status` output
- Post-sprint → suggest `/forge:retro`

| Field | Action |
|---|---|

## Checklist for upgrading

1. **Upgrade the plugin:**
   ```bash
   /plugin install forge@skillforge
   ```

2. **Run the migration:**
   ```bash
   /forge:update
   ```
   This applies migration entries: removes `config.mode`, updates generated command wrappers.

3. **Rebuild commands:**
   ```bash
   /forge:rebuild commands
   ```
   Regenerates `.claude/commands/` with the new command names.

4. **Update any scripts or notes** that reference old command names. See the rename and removal tables above.

5. **Run health:**
   ```bash
   /forge:health
   ```
   Verify no drift was introduced by the migration.

## Breaking change summary

| Breaking change | Type | Workaround |
|---|---|---|
| 7 commands renamed | Command rename | Old names return redirect error with new name |
| 6 commands removed | Command removal | Each has a replacement (see table above) |
| `config.mode` removed | Config field removed | Removed automatically by `forge:update` |
| `--fast` / `--full` flags removed | Flag removal | No replacement; all inits are now full |
| `ensure-ready.cjs` deleted | Tool removal | Not user-facing |
| `lazy-materialize.md` deleted | Workflow removal | Not user-facing |
