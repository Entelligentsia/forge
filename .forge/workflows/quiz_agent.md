# Workflow: Quiz Agent (Forge Knowledge Check)

## Purpose

Verify that an agent has correctly loaded and understood the knowledge base
before beginning a high-stakes task.

---

## Questions

1. What is the no-npm rule and where is it documented?
2. When is a version bump required? Name three criteria.
3. What must happen before a change to `forge/` can be pushed?
4. What does `node forge/tools/validate-store.cjs --dry-run` verify?
5. What is the difference between `forge/` and `engineering/` in this repo?

## Pass Criteria

All 5 questions answered correctly and specifically.
Vague answers ("generally security things") fail.

## Fail Action

Re-read `engineering/architecture/stack.md`, `engineering/architecture/processes.md`,
and `engineering/stack-checklist.md`, then retry.
