# FORGE-S11-T03: Fix orchestrate_task ROLE_TIER model fallback not applied (#57)

**Sprint:** FORGE-S11
**Estimate:** M
**Pipeline:** default

---

## Objective

When the `ANTHROPIC_DEFAULT_PRIMARY`, `ANTHROPIC_DEFAULT_SECONDARY`, and `ANTHROPIC_DEFAULT_FAST_MODEL` environment variables are unset, the orchestrator (`meta-orchestrate.md`) neither falls back to ROLE_TIER defaults nor displays the resolved model in phase announcements. The pipeline stalls before spawning subagents.

Fix `meta-orchestrate.md` so each phase resolves its model from ROLE_TIER defaults when env vars are absent, and displays the resolved model name in every phase announcement line.

## Acceptance Criteria

1. When no cluster env vars are set, each phase resolves its model from ROLE_TIER defaults:
   - `sonnet` tier → `claude-sonnet-4-6`
   - `opus` tier → `claude-opus-4-5`
   - `haiku` tier → `claude-haiku-4-5`
2. Resolved model name appears in every phase announcement line.
3. Workflow generates downstream agents correctly without stalling.
4. The fix is surgical — only the model resolution and announcement sections change; no phase structure alterations.
