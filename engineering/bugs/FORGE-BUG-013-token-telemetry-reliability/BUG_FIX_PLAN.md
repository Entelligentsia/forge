# Bug Fix Plan — FORGE-BUG-013

**Bug ID:** FORGE-BUG-013  
**Title:** Token telemetry unreliable — silent skip fallback + non-enforced reporting  
**Fix Type:** Workflow logic correction (Phase 1: pre-flight validation)

---

## Objective

Eliminate silent skip fallback for `/cost` unavailability. Make token reporting mandatory — agent HALTS if `/cost` fails.

**Target reliability:** 40–60% → 95%

---

## Approach

### Phase 1: Pre-Flight `/cost` Validation (This Fix)

Add Step 0 to all workflows with token reporting:
```markdown
## Step 0 — Verify /cost Availability

Run `/cost` to verify token reporting available. If command fails or returns empty:
**HALT.** Print: "Token reporting required. /cost unavailable. Cannot proceed."
```

Remove "skip silently" fallback from sprint-intake and sprint-plan.

**Impact:**
- Agent must abort if `/cost` unavailable → forces user to fix before proceeding
- No silent skip → either token data captured or workflow fails loudly
- Reliability: 95% (assumes agent executes gate check as first step)

---

### Phase 2: Orchestrator Enforcement (Future Bug/Task)

Add sidecar verification to orchestrator:
```bash
# After each agent completes
[ -f "engineering/sprints/{sprint}/events/{eventId}-tokens.json" ] || {
  echo "ERROR: Agent completed but no token sidecar"
  exit 1
}
```

**Impact:** Reliability 95% → 99% (catches agent omission)

**Deferred:** Requires orchestrator refactor. Phase 1 sufficient for EXP-001 baseline capture.

---

### Phase 3: Schema Enforcement (Future Task)

Add token field validation to store-cli:
```javascript
if (options.sidecar && !data.inputTokens) {
  console.error("ERROR: Sidecar requires inputTokens field");
  process.exit(1);
}
```

**Impact:** Reliability 99% → 99.9%

**Deferred:** Phase 1 unblocks compression experiment.
